**File:** `artifacts/mobile/app/(tabs)/profile.tsx`

which does the OPPOSITE of a reset — it marks onboarding as done. It never

## BUG 2 — Workout Summary Screen Gets Stuck / Shows Wrong Data
**File:** `artifacts/mobile/app/workout/summary.tsx`

**Problem:** The summary screen reads `sessions[0]` from UserContext to get
the last session. But `router.replace('/workout/summary')` is called in
`active.tsx` immediately after `await addSession(session)`. Due to React's
async state batching, `sessions[0]` in UserContext may not yet reflect the
newly saved session by the time SummaryScreen mounts. The screen sometimes
shows nothing or the wrong previous session.

**Fix:** Pass the completed session data directly via router params instead
of reading from context state.

In `artifacts/mobile/app/workout/active.tsx`, change the navigation call
inside `finishWorkout()`:

```ts
// BEFORE:
await addSession(session);
cancelWorkout();
router.replace('/workout/summary');

// AFTER:
await addSession(session);
cancelWorkout();
router.replace({
  pathname: '/workout/summary',
  params: { sessionId: session.id },
});
```

In `artifacts/mobile/app/workout/summary.tsx`, read from both params AND
context as a fallback:

```ts
import { useLocalSearchParams } from 'expo-router';

export default function SummaryScreen() {
  const { sessionId } = useLocalSearchParams<{ sessionId: string }>();
  const { sessions } = useUser();

  // Find by ID first, fall back to most recent
  const lastSession = sessionId
    ? sessions.find((s) => s.id === sessionId) ?? sessions[0]
    : sessions[0];
  // ... rest of component unchanged
```

## BUG 7 — AI Generate Plan Response Parsing Too Fragile
**File:** `artifacts/api-server/src/routes/ai.ts`

**Problem:** The `/generate-plan` endpoint uses `text.match(/\{[\s\S]*\}/)` to
extract JSON. If Claude wraps the response in markdown code fences
(` ```json ... ``` `) or adds any explanation text, this regex will fail to
extract the correct JSON, causing a 500 error. There is also no try-catch
around `JSON.parse()`, so a malformed JSON response crashes the endpoint.

**Fix:** Replace the response parsing block in the `/generate-plan` route:

```ts
// BEFORE:
const text = data.content?.[0]?.text ?? '';
const jsonMatch = text.match(/\{[\s\S]*\}/);

if (jsonMatch) {
  const parsed = JSON.parse(jsonMatch[0]) as { plan_name: string; workouts: unknown[] };
  res.json(parsed);
} else {
  res.status(500).json({ error: 'Could not parse AI response' });
}

// AFTER:
let text = data.content?.[0]?.text ?? '';

// Strip markdown code fences if present
text = text.replace(/^```(?:json)?\s*/m, '').replace(/\s*```\s*$/m, '').trim();

// Extract the JSON object
const jsonMatch = text.match(/\{[\s\S]*\}/);

if (!jsonMatch) {
  res.status(500).json({
    error: 'AI did not return valid JSON',
    preview: text.slice(0, 300),
  });
  return;
}

try {
  const parsed = JSON.parse(jsonMatch[0]) as { plan_name: string; workouts: unknown[] };
  if (!parsed.workouts || !Array.isArray(parsed.workouts)) {
    res.status(500).json({ error: 'AI response missing workouts array' });
    return;
  }
  res.json(parsed);
} catch (parseErr) {
  res.status(500).json({
    error: 'JSON parse failed',
    preview: jsonMatch[0].slice(0, 300),
  });
}
```

## BUG 8 — Active Workout: completeSet Does Not Trigger Rest Timer
**File:** `artifacts/mobile/app/workout/active.tsx`

**Problem:** In the `ExerciseBlock` component (rendered for each exercise),
the `onCompleteSet` callback calls `completeSet(exIdx, setIdx)` from
WorkoutContext. This toggles the set's completed state correctly. BUT it never
calls `startRestTimer(exIdx)`. The rest timer in WorkoutContext is never
started after a set is completed. Users complete sets and the rest timer
bottom sheet never appears.

**Find this line in the ExerciseBlock's onCompleteSet handler:**
```ts
const onCompleteSet = (exIdx: number, setIdx: number) => {
  completeSet(exIdx, setIdx);
};
```

**Replace with:**
```ts
const onCompleteSet = (exIdx: number, setIdx: number) => {
  completeSet(exIdx, setIdx);
  // Only start rest timer when completing (not un-completing) a set
  // We know it's completing if the set was NOT completed before
  const wasCompleted = activeSession?.exercises[exIdx]?.sets[setIdx]?.completed;
  if (!wasCompleted) {
    const restDuration = profile?.restDuration ?? 90;
    startRestTimer(exIdx, restDuration);
  }
};
```

Note: `profile` is already destructured from `useUser()` in this component.
`startRestTimer` is already destructured from `useWorkout()`.
This fix just needs to call `startRestTimer` after `completeSet`.

## BUG 9 — Active Workout: Exercise Blocks Are Missing `muscleGroupColors` Variable
**File:** `artifacts/mobile/app/workout/active.tsx`

**Problem:** The `ExerciseBlock` sub-component uses `muscleColor` variable
which comes from `MUSCLE_GROUP_COLORS[exercise.muscleGroup]`. If a muscle
group is not found in the map, `muscleColor` is `undefined`. Then
`backgroundColor: muscleColor + "22"` becomes `"undefined22"` which is an
invalid color string. This causes a React Native style warning and the muscle
group badge renders with a broken background.

**Fix:** Add a fallback in ExerciseBlock:
```ts
// BEFORE (find this line):
const muscleColor = MUSCLE_GROUP_COLORS[exercise.muscleGroup];

// AFTER:
const muscleColor = MUSCLE_GROUP_COLORS[exercise.muscleGroup] ?? '#888888';
```
